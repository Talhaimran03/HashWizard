Password Salting


Il "salting" delle password è una tecnica di sicurezza utilizzata per proteggere le password dagli attacchi di forza bruta e dai tentativi di decodifica. In sostanza, il salting consiste nell'aggiungere un valore casuale (noto come "salt") alla password prima di eseguire la funzione di hash.
Tuttavia, i criminali informatici possono sfruttare il fatto che le persone utilizzano spesso password comuni o facili da indovinare. Gli hacker possono creare un "dizionario" di hash pre-calcolati per le password comuni e confrontarli con gli hash rubati. Se la password originale è presente nel dizionario, l'hacker può scoprire la password corrispondente.
Il salting delle password rompe questa corrispondenza. Quando un salt viene aggiunto alla password prima dell'hashing, anche se due utenti hanno la stessa password, i loro hash saranno diversi a causa del valore del salt unico. Ogni utente avrà un salt diverso, quindi anche se le password sono uguali, gli hash saranno diversi.

L'uso del salt rende anche più difficile l'utilizzo di attacchi di tabulazione dei rainbow, che sono basati su un dizionario pre-calcolato di hash di password comuni. Anche se un attaccante crea un dizionario di hash per le password più comuni, senza conoscere il salt, il dizionario non sarà utile per violare le password. Inoltre, il salt viene memorizzato insieme all'hash della password nel database dell'applicazione. Quando un utente tenta di accedere, il salt associato al suo account viene estratto dal database e combinato con la password inserita. Il risultato viene quindi hashato e confrontato con l'hash memorizzato nel database. Solo se corrispondono, allora la password è considerata corretta.

In conclusione, il salting delle password è una tecnica portante per migliorare la sicurezza delle password memorizzate. Aggiungendo un valore casuale (salt) alle password prima dell'hashing, si rende molto più difficile per gli attaccanti decifrare le password rubate.


Hashing


L'hashing è un algoritmo utilizzato per convertire dati di qualsiasi dimensione in una stringa di lunghezza fissa, nota come hash. Gli algoritmi di hash sono progettati in modo tale che il risultato dell'operazione di hashing sia univoco per ogni input diverso. Inoltre, un cambiamento anche minimo nell'input produrrà un hash completamente diverso.

L'hashing viene utilizzato in sicurezza informatica per diverse ragioni:

1. Verifica dell'integrità dei dati: L'hashing può essere utilizzato per verificare se un dato è stato alterato o corrotto. Ad esempio, se si desidera inviare un file e si calcola l'hash del file prima e dopo l'invio, è possibile confrontare i due hash per assicurarsi che il file sia stato trasmesso correttamente senza modifiche;

2. PROTEZIONE DELLA PASSWORD: Le password sono spesso memorizzate come hash anziché in formato testo semplice. Quando un utente crea una password, viene eseguita l'operazione di hashing sull'input della password e l'hash risultante viene memorizzato nel database. Quando l'utente tenta di accedere, la password inserita viene hashata nuovamente e confrontata con l'hash memorizzato. Se i due hash corrispondono, l'accesso viene consentito. Questo fornisce un livello di sicurezza aggiuntivo, poiché anche se l'hash viene compromesso, non è possibile risalire alla password originale. Nel salting precedentemente spiegato viene utilizzato proprio per questo motivo. Rendere sicure le password.;

3. Firma digitale: Le firme digitali sono utilizzate per garantire l'autenticità e l'integrità di un messaggio o di un documento elettronico. Una firma digitale viene creata eseguendo l'hash del messaggio o del documento e applicando una crittografia asimmetrica utilizzando una chiave privata. La firma digitale può essere verificata utilizzando la chiave pubblica corrispondente, garantendo così l'autenticità del mittente e l'integrità dei dati;
